# ✨ 工作安全守则

本文件中的所有规则，我（GitHub Copilot）在每次执行任务前都会**自动读取并遵守**。

---

## 第一优先：每次修改前先提交

> **无论你要做什么修改，第一步永远是提交当前代码！**

```powershell
cd <项目根目录>
git add -A
git commit -m "backup: <修改简述，Y-m-d>"
git push
```

- 修改了文件后发现需要回滚 → **我全责**
- **没有例外，必须先 commit 再干活。**

---

## 第二优先：操作前先理解，不盲动

1. **不确定就先问** —— 如果我不确定某段代码的作用，必须先问清楚再动手，不瞎猜。
2. **完整阅读再动** —— 至少要读完整函数/文件，不只看开头几行就下结论。
3. **改后检查全链路** —— 前端改了 UI 就确认后端 API 也对得上；改了 API 就确认调用方也更新了。

---

## 禁止操作

- ❌ **删除目录或文件前必须确认** —— 任何删除操作前，必须确认没有其他文件依赖它。
- ❌ **覆盖大段代码前必须先备份** —— 修改超过 20 行的函数时，先确认已 git commit 或另有备份。
- ❌ **替换整个目录** —— 不要用一个目录替掉另一个目录然后再"补"文件回来。逐文件对比合并。
- ❌ **修改不了解的链路** —— 不知道完整调用链路的代码，必须先问清楚再动。
- ❌ **使用非项目指定的环境** —— 不要用系统 Python 或其他未指定的环境执行项目操作。
- ❌ **假设文件无用就删除** —— 判断"废弃"前，至少全项目搜索是否有引用。
