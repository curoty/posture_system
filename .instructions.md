## 第一优先：每次修改前先提交（保护现场）

> **无论你要做什么修改，第一步永远是提交当前代码！**

### 操作流程

```powershell
# Step 1 - 提交当前工作区所有变更
cd C:\Users\xunyi\Desktop\posture-system
git add -A
git commit -m "backup: <本次修改的简短描述，Y-m-d>"

# Step 2 - 推送到远端（可选但推荐）
git push

# Step 3 - 然后才开始做你要的修改
```

### 违反此规则的后果

- 修改了文件后发现需要回滚 → **我全责**
- 前端/后端代码被我改出 bug → **无备份则无法恢复**
- **必须、必须、必须**先 commit 再干活，没有例外。
