# IMU 前两阶段优化工作简要说明

日期：2026-07-20  
硬件：ICM20602 六轴 IMU、ESP32-S3 SuperMini  
当前节点：腰部、左膝、右膝、左脚踝、右脚踝，共 5 节点

## 一、总体目标

本轮工作主要完成：

1. 修复五节点陀螺仪零偏校准机制；
2. 打通训练数据的采集、通信、原始存储、软件处理和派生数据存储链路；
3. 增加时间戳、丢帧、节点覆盖率、五节点对齐和异常尖峰处理；
4. 为后续原始、仅校准、校准加滤波三组消融实验准备统一的数据结构。

## 二、阶段一：零偏机制

### 2.1 五节点测试固件

已修改腰部、左膝、右膝、左脚踝和右脚踝五份测试固件。

完成内容：

- 取消上电后无条件接受 100 帧陀螺仪均值；
- 取消自动启动校准，改为训练页面主动触发；
- 校准前检查设备是否静止；
- 检查陀螺仪模长和三轴标准差；
- 检查加速度模长是否接近 1 g；
- 检查加速度模长标准差；
- 校准失败时不覆盖已有有效 offset；
- 重新校准成功时替换当前临时 offset；
- 支持清除 RAM 中的临时 offset；
- 临时 offset 断电后自然失效，不写入永久存储；
- 保留 ICM20602 硬件 DLPF；
- WiFi 连接失败后支持后台重连。

### 2.2 校准状态上报

固件通过 MQTT 上报：

- `calibration.status`
- `calibration.sample_count`
- `calibration.gyro_offset`
- `calibration.gyro_std_max`
- `calibration.acc_norm_mean`
- `calibration.acc_norm_std`
- `calibration.temperature_c`
- `filter_status`

### 2.3 训练页面主动校准

主动校准功能放置在训练数据采集页面，而不是静止采集测试页面。

完成内容：

- 校准已选择节点；
- 校准前 3 秒倒计时；
- 向选中节点发送校准命令；
- 轮询并显示各节点校准结果；
- 新进入训练页面时清除上一会话临时 offset；
- 校准失败节点不会伪装成校准成功；
- 保存训练样本时携带本次校准元数据。

### 2.4 Python 校准语义

区分三类参数：

1. 设备基准 offset：实验台长期标定参数；
2. 本次主动校准 offset：固件本次有效静止校准得到并已经扣除的参数；
3. 残余偏置：固件校准后仍然存在、需要 Python 显式补偿的小偏置。

固件已经扣除的 offset 记录在：

```text
calibrationProfiles.metadata.device_applied_gyro_offset
```

Python 的 `gyro_bias` 默认保持为零，避免重复扣除固件 offset。只有明确提供有效残余偏置时，Python 才继续校准。

## 三、阶段二：数据质量与滤波

### 3.1 五节点时间对齐

训练采集页面现在明确请求：

```text
assembleNodes = true
assemblyToleranceMs = 25
```

采集流程：

```text
获取服务器当前时间
→ 建立本次采集开始边界
→ 排除 MQTT 缓存中的历史帧
→ 按选中节点进行时间匹配
→ 按时间戳去重
→ 达到目标对齐帧数后返回
```

如果目标为 180 帧但实际不足，采集会返回明确错误，不把不完整数据当作成功样本。

### 3.2 时间戳和数据质量检查

Python 已实现：

- 重复时间戳检测；
- 时间戳倒退检测；
- 采样间隔统计；
- 时间间隔抖动统计；
- 时间缺口检测；
- 估计丢失帧数；
- 每节点样本数统计；
- 每节点缺失帧数统计；
- 每节点覆盖率统计；
- 五节点完整性报告。

### 3.3 异常值与尖峰

已实现：

- 加速度物理范围检查；
- 陀螺仪物理范围检查；
- 基于中位数和 MAD 的稳健异常检测；
- 对孤立异常点和尖峰进行替换；
- 统计各节点及全部节点的尖峰替换数量。

### 3.4 DLPF 和软件低通

- ICM20602 硬件 DLPF 保持开启；
- 当前配置约为陀螺仪 20 Hz、加速度计 21.2 Hz；
- Python 软件低通已实现；
- 加速度计和陀螺仪支持不同截止频率；
- 软件低通默认关闭；
- 等真实动作数据到位后进行 A/B 验证，再决定是否启用；
- 温漂补偿保持关闭，但保留温度和校准温度记录。

### 3.5 数据格式兼容

软件滤波同时兼容两种训练帧：

```json
{"t": 20, "points": {"waist": {"ax": 0, "ay": 0, "az": 1, "gx": 0, "gy": 0, "gz": 0}}}
```

```json
{"t": 20, "p": {"waist": [0, 0, 1, 0, 0, 0]}}
```

输出保持输入结构，避免数据库、云函数和模型服务之间出现格式不一致。

## 四、训练数据采集与存储链路

当前设计链路：

```text
五节点 ICM20602
→ ESP32-S3
→ MQTT
→ FastAPI /frames
→ 五节点时间对齐
→ 小程序训练数据采集页面
→ skateActionAnalyze 云函数
→ train_samples_nofiltering
→ FastAPI /denoise/training-frames
→ train_samples_filtering
```

### 4.1 原始集合

```text
train_samples_nofiltering
```

规则：

- 原始样本优先保存；
- 原始样本只写入，不被软件滤波覆盖；
- 保存动作标签、选中节点、校准元数据和原始帧；
- 滤波服务失败不删除原始样本。

### 4.2 派生集合

```text
train_samples_filtering
```

同一原始样本可生成：

- `processing_variant = calibrated`
- `processing_variant = calibrated_filtered`

通过以下字段关联：

```text
source_sample_id
processing_variant
processing_version
processing_config
quality_report
```

重试同一处理版本时更新已有记录，不重复生成同版本数据。

### 4.3 三组消融数据

1. 原始组：直接使用 `train_samples_nofiltering`；
2. 仅校准组：残余校准，不处理尖峰，不做软件低通；
3. 校准加滤波组：残余校准、尖峰处理、可选软件低通。

不需要额外建立 `train_samples_only_calibrate` 集合。

## 五、接口和云函数

FastAPI 新增：

```text
POST /calibration/gyro
POST /calibration/gyro/clear
POST /denoise/training-frames
```

云函数新增：

```text
processSensorTrainingSample
```

保存原始训练样本后，云函数自动尝试生成仅校准和校准加滤波两个版本。处理失败时，原始样本仍然保留，并记录 `processingStatus = partial_failed`，之后可按 `sample_id` 重试。
