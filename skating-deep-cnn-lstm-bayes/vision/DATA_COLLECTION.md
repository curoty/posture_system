# 视觉数据采集与标注指南

## 概述

从零开始采集花样滑冰视频数据，用于训练 ST-GCN 动作分类和光质量评分模型。
目标是采集"**视频 + IMU 同步录制**"的配对数据，为未来的多模态融合做准备。

---

## 第一阶段: 纯视频采集 (立即开始)

### 拍摄要求

| 项目 | 建议 |
|------|------|
| 相机 | 手机 (30fps 以上) 或任何摄像头 |
| 分辨率 | 720p 以上 |
| 帧率 | **30fps 固定** (与推理管道的 fps 配置一致) |
| 视角 | **正面** 或 **侧面**, 固定机位 |
| 背景 | 纯色/简单背景, 避免多人入镜 |
| 光照 | 充足均匀, 避免背光 |
| 时长 | 每次动作 5-15 秒, 包含完整动作 + 前后各 1 秒缓冲 |

### 一个动作录一个视频

每段视频只包含**一个完整动作**。文件命名格式:

```
{动作名称}_{日期}_{序号}.mp4

例如:
weight_shift_20260708_001.mp4
weight_shift_20260708_002.mp4
```

如果录制的是"非指定动作"(比如过渡动作、杂乱片段)，文件名用 `other_xxx.mp4`。

### 最小数据量目标

| 阶段 | 数量 | 用途 |
|------|------|------|
| 起步测试 | 5-10 段 | 跑通 RTMPose 提取管道 |
| 模型训练 | ≥100 段 | ST-GCN 初训 |
| 生产可用 | ≥500 段 | + LightGBM 质量评分稳定 |

## 第二阶段: 视频 + IMU 同步采集 (为多模态融合准备)

这是**关键步骤**——目的是让每条视频都有对应的 IMU 数据，时间轴对齐。

### 同步方案 (不需要专业设备)

**方法: "三闪对齐法"**

1. 把 IMU 设备 (或手机 App 模拟的 IMU 信号) 放在镜头前
2. 录制开始时，**用手在镜头前快速挥 3 下** (或让 IMU 设备的 LED 闪 3 次)
3. 这个"三闪"是你后期对齐两个模态的时间锚点
4. 然后让运动员正常做动作

### 后期对齐操作

```
视频帧:    [三闪] [动作开始] [动作过程] [动作结束]
            ↑                        ↑
            t=0 (对齐起点)          t=4 (窗口结束)

IMU 帧:    [三闪] [动作开始] [动作过程] [动作结束]
            ↑                        ↑
            t=0 (对齐起点)          t=4 (窗口结束)
```

对齐脚本 `tools/align_modalities.py` (后续编写) 可以自动完成:
1. 在视频帧中检测"手部快速挥动"事件 → 标记为 t=0
2. 在 IMU 信号中检测"信号突变"事件 → 标记为 t=0
3. 计算两者的时间偏移量，校正所有时间戳

### 数据目录结构

```
data/
├── videos/                     # 原始视频
│   ├── weight_shift_001.mp4
│   └── weight_shift_002.mp4
├── imu_raw/                    # 同步的 IMU 原始数据
│   ├── weight_shift_001.jsonl
│   └── weight_shift_002.jsonl
├── keypoints/                  # RTMPose 提取后的关键点
│   ├── weight_shift_001.jsonl
│   └── weight_shift_002.jsonl
├── labels.csv                  # 标注表 (见下文)
└── paired_index.json           # 视频↔IMU 配对索引
```

## 标注规范

### 需要标注的字段

每段视频需要人工标注以下信息:

| 字段 | 格式 | 示例 | 说明 |
|------|------|------|------|
| `video_name` | string | `weight_shift_001.mp4` | 视频文件名 |
| `action_label` | string | `weight_shift` | 动作类别 |
| `quality_score` | float [0-100] | `82.5` | **质量评分** (最重要!) |
| `quality_level` | string | `良好` | 质量等级 (自动从分数映射) |
| `has_imu` | bool | `true` | 是否有同步 IMU 数据 |
| `imu_file` | string or null | `weight_shift_001.jsonl` | 对应 IMU 文件名 |
| `notes` | string | `左膝角度偏大` | 备注 |

### 质量评分标注参考

| 分数区间 | 等级 | 参考描述 |
|---------|------|---------|
| 88-100 | 优秀 | 动作标准流畅, 姿态控制好 |
| 75-87 | 良好 | 动作基本正确, 有少量瑕疵 |
| 60-74 | 一般 | 动作框架对但细节差, 需要改进 |
| 0-59 | 不合格 | 动作变形, 重心不稳或未完成 |

### 标注工具

初期直接用 Excel/CSV 管理。推荐格式:

```csv
video_name,action_label,quality_score,quality_level,has_imu,imu_file,notes
weight_shift_001.mp4,weight_shift,82.5,良好,true,weight_shift_001.jsonl,重心偏右
weight_shift_002.mp4,weight_shift,90.0,优秀,false,,
```

## 采集流程 (完整步骤)

### 第 1 步: 设置手机

1. 打开相机 → **锁定帧率 30fps** (专业模式或第三方 App)
2. 固定在三脚架上
3. 确保视野覆盖全身

### 第 2 步: 开始录制

1. IMU 设备开机 (如需同步录制)
2. 在镜头前完成"三闪对齐信号"
3. 运动员进入画面, 做动作
4. 录制 5-15 秒
5. 停止录制

### 第 3 步: 数据管理

1. 按命名规则重命名文件
2. 复制到 `data/videos/`
3. 更新 `labels.csv` 填写标注
4. (可选) 运行 `python -m vision.pose_extractor --extract data/videos/xxx.mp4` 提取关键点

### 第 4 步: 数据验证

运行验证脚本检查数据完整性:

```bash
python tools/validate_vision_data.py --data-dir data/
```
