### 1 个 FastAPI 实例，6 个路由

`src/sensor_api.py` 是唯一入口。`src/api.py` 保留为独立推理服务（可单独启动，端口 18081），但生产不用。

| 路由 | 方法 | 说明 |
|---|---|---|
| `/health` | GET | MQTT 连接状态、模型是否加载 |
| `/frames` | POST | 实时传感器帧轮询（支持 `assembleNodes` 多节点对齐） |
| `/calibration/gyro` | POST | 仅对 `get_active_roles()` 检测到的真实在线节点发送校准命令 |
| `/calibration/gyro/clear` | POST | 清除固件 RAM 临时 offset（新采集会话） |
| `/denoise/training-frames` | POST | 尖峰检测 + 低通滤波，输入/输出保持 `t`/`p` 结构 |
| `/infer` | POST | 云函数兼容推理 — 直接调用 CNN-LSTM + LightGBM |

### 模型加载

`sensor_api.py` 启动时尝试加载两个模型。任一加载失败都不阻塞服务启动，只有 `/infer` 会返回 503。

| 模型 | 路径 | 用途 |
|---|---|---|
| CNN-LSTM 动作分类器 | `experiments/weight_shift_v2/action_model.pt` | 9 节点 IMU → 5 种动作 |
| LightGBM 质量评分 | `experiments/lgb_quality_v3/lgb_quality_model.pkl` | 特征 → 连续 0-100 分数 |

### 校准链路

```text
前端 3 秒倒计时 → POST /calibration/gyro (roles=选中节点)
  → get_active_roles() 扫描 MQTT 缓冲区（30s 内有效数据的节点）
  → 交集过滤（仅校准真实在线节点）
  → publish_gyro_calibration_command() 发 MQTT 命令
  → 固件 calibrateGyroscope() — 3 秒静止检测
      ├─ 合格 → gyroCalibrationValid=true，MQTT 帧 status="ready"
      └─ 不合格 → gyroCalibrationValid=false，MQTT 帧 status="invalid"
  → 前端等 5 秒后轮询 /frames（8 轮 × 1秒），读取 frame.calibration.status
  → 保存 calibrationProfiles，在保存样本时传给云函数
```

### 固件关键行为

- **上电不自动校准**，等 MQTT 命令（`"startup calibration disabled; waiting for UI command"`）
- **主动校准触发**：接收 MQTT topic `sensor/imu/commands/{role}`，`"calibrate_gyro"`
- **静止判断**：4 个条件（`gyro_norm ≤ CAL_MAX_GYRO_NORM_DPS`、`gyro_std_max ≤ CAL_MAX_GYRO_STD_DPS`、`acc_norm 范围 [0.9, 1.1]g`、`acc_norm_std ≤ CAL_MAX_ACCEL_NORM_STD_G`）
- **合格**：`gyroCalibrationValid=true`，MQTT 帧 `"status":"ready"`
- **不合格**（2026-07-23 修复）：`gyroCalibrationValid=false`，offset 清零，**不再恢复旧值**，MQTT 帧 `"status":"invalid"`。让 Python `denoise.py` 走 bench offset
- `/calibration/gyro/clear` → `gyroCalibrationValid=false`，offset 清零
- MQTT 帧上报 8 个字段：`status`、`sample_count`、`gyro_offset`、`gyro_std_max`、`acc_norm_mean`、`acc_norm_std`、`temperature_c`、`filter_status`

## 九、数据链路全景

### 训练数据采集

```text
ICM20602（5 节点，硬件 DLPF 约 20 Hz）
  → ESP32-S3（50 Hz 采样）
  → MQTT Broker 82.156.18.205:1883
  → skating-deep-cnn-lstm-bayes/src/mqtt_client.py（帧缓冲）
  → FastAPI /frames（assembleNodes=true, assemblyToleranceMs=25）
  → device-sensor-adapter.js（去重、对齐）
  → debug.js（采集、标注）
  → sensor-model.js → skateActionAnalyze 云函数
  → train_samples_nofiltering（原始集合，不可覆盖）
  → FastAPI /denoise/training-frames（尖峰 + 可选低通）
  → train_samples_filtering（派生集合）
```

### 主动校准

```text
debug.js[onTapCalibrateSelectedNodes]
  → 弹窗确认 → 3 秒倒计时
  → POST /calibration/gyro(roles)
  → get_active_roles() 过滤在线节点
  → publish_gyro_calibration_command(roles) → MQTT → 固件
  → 固件 calibrateGyroscope()
      ├─ 合格 → gyroCalibrationValid=true, 帧 status="ready"
      └─ 不合格 → offset=0, gyroCalibrationValid=false, 帧 status="invalid"
  → 前端等 5 秒 → 轮询 /frames（8 轮 × 1 秒）
  → 读取 frame.calibration.status
  → calibrationProfiles[role] = {...}
  → 保存样本时传给 skateActionAnalyze
```

### AI 推理

```text
小程序 → remote-predict.js → predictAnalysis 云函数
  → HTTP POST → FastAPI /infer
  → _payload_to_record() 格式转换
  → predict_record() → CNN-LSTM + LightGBM
  → _deep_result_to_analysis() 格式转换
  → 返回 analysis
```

### 用户登录
